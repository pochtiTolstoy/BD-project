# Предметная область для проекта

## Общее описание

Объект "Человек" рассматривается в контексте покупателя маркетплейса техники.

Модель предметной области описывает не весь маркетплейс, а только данные о человеке как о покупателе. Товары, заказы, корзина, складские остатки и платежи не входят в рассматриваемую область.

## Категории данных о человеке

Ниже указаны не таблицы и не структура БД, а смысловые группы данных. Примеры внутри групп не являются жестко фиксированным списком полей.

### Базовые персональные данные

Категория описывает основные анкетные сведения о человеке.

Примеры данных:

- ФИО;
- дата рождения;
- пол.

### Данные аккаунта покупателя

Категория описывает учетную запись человека в маркетплейсе.

Примеры данных:

- логин или идентификатор аккаунта;
- статус аккаунта;
- дата регистрации;
- информация об активности аккаунта, если требуется.

### Контактные данные

Категория описывает способы связи с человеком.

Примеры данных:

- email;
- телефон;
- мессенджер или другой дополнительный канал связи.

### Адресные данные

Категория описывает адреса, которые могут использоваться для доставки или связи с покупателем.

Примеры данных:

- адрес доставки;
- страна;
- регион;
- город;
- улица;
- дом;
- квартира;
- почтовый индекс;
- комментарий к доставке;
- исходная строка адреса, если адрес хранится в свободном формате.

### Документы для опциональной верификации

Категория описывает документ, который может использоваться для проверки личности.

Примеры данных:

- тип документа;
- серия или номер документа;
- дата выдачи;
- кем выдан;
- статус проверки документа.

Документы не являются обязательными для каждого покупателя, но могут использоваться для проверки личности, покупки дорогой техники, рассрочки или спорных возвратов.

### Согласия пользователя

Категория описывает факт согласия или отказа пользователя на определенное действие.

Внутри одного согласия могут быть такие данные:

- тип согласия;
- признак, дано согласие или нет;
- дата выдачи согласия;
- дата отзыва согласия, если оно было отозвано;
- источник получения согласия.

Примеры типов согласий:

- обработка персональных данных;
- рекламные рассылки;
- передача данных партнерам.

### Дополнительные признаки покупателя техники

Категория описывает гибкие покупательские признаки, которые могут отличаться от системы к системе и не всегда обязательны для каждого человека.

Примеры данных:

- любимая категория техники;
- предпочитаемый бренд;
- интерес к рассрочке;
- примерный бюджет покупки;
- наличие устройств умного дома;
- предпочитаемая экосистема устройств;
- уровень лояльности или клиентский сегмент.

## Ограничения согласования

Для сохранения смысла задания по миграции заранее не согласовываются:

- структура таблиц;
- названия таблиц;
- названия полей;
- типы данных в БД;
- связи между таблицами;
- точный формат выгрузки;
- правила маппинга.

Согласуется только общая предметная область и смысловые категории данных.

## Краткое резюме

Предметная область: человек как покупатель маркетплейса техники.

Основные данные: профиль человека, аккаунт покупателя, контакты, адреса доставки, опциональные документы для верификации, согласия и дополнительные покупательские признаки.
